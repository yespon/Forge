# 企业 Agent Runtime 平台 — 需求清单

## 核心定位

多用户 Skill/Plugin Runtime 平台，非工作流编排（非 Dify 模式）。它借鉴 Codex 的端云一体、持续会话、任务运行时、工具调用、审批和产物管理体验，但默认目标不是 coding，而是通过企业 Skill、Plugin、MCP 和第三方连接器完成个人与企业任务。

每个用户拥有持久化的 agent 会话环境和任务历史，可以安装/调用 public、team、private skills。bash、文件读写、代码编辑等能力仅作为受限内置 Skill 或特定插件存在，不作为所有 Agent 的默认能力。

---

## 设计原则

1. Skill/Plugin 优先：Agent 先解析可用 Skill、权限、凭证和运行时要求，再执行任务。
2. Task Runtime 优先：聊天只是任务发起、澄清和控制界面；长程、异步、定时、审批暂停都由 Task 承载。
3. 权限传导优先：第三方和企业系统调用必须绑定用户身份、scope、审批策略和审计日志。
4. Sandbox 是隔离边界：用于插件执行、依赖隔离、临时文件、网络策略和产物生成，不等同于 coding workspace。
5. HITL 是治理主路径：跨系统写操作、批量变更、外发消息、敏感数据访问都应能触发审批。

## 需求列表

### 1. Agent Runtime

一个运行 skills/plugins/connectors 的 agent runtime。底层可使用 LangGraph/DeepAgents，但平台层抽象是 Skill 和 Task，而不是默认 bash/file tools。

### 2. 多用户支持

支持多用户，用户间隔离。

### 3. Sandbox 环境

按需创建和销毁，原则上按 task/session/plugin 隔离。用于插件运行、依赖隔离、临时文件、环境变量注入、网络策略和产物生成。

### 4. 权限隔离

第三方权限传导及隔离（例如飞书操作权限）。

### 5. Skill 隔离

公有 skill 共享，团队 skill 部分共享，私有 skill 不共享。Skill Registry、安装状态、版本、权限声明、HITL 策略和运行时要求是 P0 主路径。

### 6. 可监控可观测

平台级和用户级的监控与可观测性。

### 7. 定时任务与异步任务

Agent loop 和企业系统交互可能很长，需要长程调度的异步任务作为后台任务执行。支持定时任务、等待外部回调、等待审批、失败重试和恢复。

### 8. TUI 和 GUI

支持 TUI 和 GUI，丰富的 GUI 能力。

### 9. 沙箱可配置

比如配置本地时支持 TUI 操作，拥有更多能力。

### 10. MCP 和工具注册管理

统一的 Skill/Plugin/Connector/Tool 注册与管理体系，支持 MCP 协议。所有工具调用都经过权限解析、凭证注入、HITL 检查和审计记录。

### 11. Human-in-the-Loop 审批

危险操作暂停等待确认。这也是需要支持异步任务的一个场景，可能会长时间阻断。可参考 LangGraph 的 interrupt 机制。

### 12. 并行任务支持

多用户间，每个用户都可以同时跑多个任务（可以设置上限）。

### 13. 实时流式显示

Agent 思考和工具调用的实时流式显示（开关，用户可配置）。

### 14. 四大核心视图

- 任务工作台（Agent 工作台 + Task 进度）
- Skill/Plugin 市场
- 审批与通知中心
- 管理后台

### 15. 通知系统

HITL 审批、任务完成/失败，需要告知用户。方案：平台内通知（GUI 红点）+ 可选外部通知（飞书/邮件）。

### 16. 任务模板 / Prompt 管理

常用任务可以保存为模板，避免每次重新输入。类似 Claude Code 的 CLAUDE.md / slash commands。

### 17. 输出产物管理

Agent/Skill 生成的文档、表格、报告、附件、数据结果、代码或导出包需要统一管理。Sandbox 文件系统可以暂存，但平台需要 Artifact 注册、导出和分享能力。

### 18. 任务历史与重放

查看历史任务的完整执行链，支持"基于此任务继续"。LangGraph checkpoint 原生支持，需要 UI 层暴露。

### 19. 多 Agent 协作（Sub-agent）

一个任务可以 spawn 多个子 Agent 并行执行。LangGraph 原生支持。

### 20. 多 Channel 接入

增加 Gateway，多 channels 支持，用户可以通过多种 IM 接入，进一步增强任务操作和通知系统。
